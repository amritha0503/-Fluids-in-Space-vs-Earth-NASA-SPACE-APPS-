   Q: How does your filtering work?
   A: "Two-step process: First, I check if the dataset accession 
       starts with physical science prefixes like PSI or FLU. 
       Second, I search the title and description for 14 fluid 
       mechanics keywords. Only datasets that match either criteria 
       are shown. Then I sort by relevance, putting PSI datasets 
       first since they're most relevant to the project."

   Q: Why did you add caching?
   A: "To optimize performance and reduce API calls. The cache 
       stores results for one hour, so subsequent loads are nearly 
       instant. Users can click refresh to clear the cache and fetch 
       fresh data if needed."

   Q: What happens when the API fails?
   A: "I implemented curated fallback content that provides helpful 
       guides, recommended search terms, and direct links to NASA's 
       portal. This way users always have access to useful information 
       even when browser CORS restrictions apply."
